Runbook: System Status Check

Category: system_status   Urgency: low   Target app: internal-admin-portal

When to use

Run this check before escalating any ticket that mentions "the system is down", "cannot log in", or "the app is broken". Many such reports are caused by ongoing incidents that are already known.

Procedure

  1. Navigate to /status in the Internal Admin Portal.
  2. Check the component health indicators: auth, database, mail, external identity provider.
  3. If any component shows degraded or down, note the incident ID displayed on the status page.
  4. Post the incident ID in the ticket and tell the user the issue is being tracked. Do not escalate for confirmed ongoing incidents.
  5. If all components are healthy, continue with normal troubleshooting per the user's original report.

Postconditions

Either the ticket is resolved with an incident reference, or the status check is recorded and normal triage continues.
