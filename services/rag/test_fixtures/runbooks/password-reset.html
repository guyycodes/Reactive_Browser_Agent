Runbook: Password Reset in the Internal Admin Portal

Category: password_reset   Urgency: medium   Target app: internal-admin-portal

Preconditions

  • The requester's identity must be verified. Match the ticket email header to the user record.
  • The target user must exist and be in status active or locked.
  • The requester must not have had more than 3 password resets in the past 24 hours (rate limit).

Procedure

  1. Navigate to /users in the Internal Admin Portal.
  2. Search for the user by email address using the search input.
  3. Open the matching user's detail page.
  4. Click the "Reset Password" button. A confirmation dialog appears.
  5. Review the dialog carefully. Confirm the action. A password reset email is sent automatically.
  6. Verify the success toast appears and the user record shows password_reset_at: now.

Postconditions

Update the ticket with resolution note: "Password reset completed at {timestamp}. User will receive a reset link via email."

Rollback

A password reset is not reversible. Always require a human review gate before confirming.
